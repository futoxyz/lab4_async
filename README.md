# Задачи
`src\task.py`
Task представляется через обычный класс. Любая задача имеет:
- **id** — уникальный идентификатор задачи.
- **description** — описание задачи.
- **priority** — порядок приоритета задачи. По умолчанию принимает *0*.
- **status** — текущий статус задачи. По умолчанию принимает *pending*.
- **task_type** — тип задачи для обработки.

**id**, **description**, **priority** и **task_type** проверяются через дескрипторы в том же файле.

Для **status** проверка происходит через сеттер внутри класса.

Допустимые значения статуса: *pending*, *in_progress*, *failed*, *completed*.

# Источники задач
`src\source.py`

- API
- Генератор
- Чтение из файла

Формат строки файла для записи как задача:

<айди> <описание>

# Очередь задач
`src\queue.py`

TaskQueue — класс с поддержкой итерации базовых методов Python. Методы очереди:
- **add** — Добавить задачу.
- **get** — Достать задачу из очереди.
- **filter_by_priority** — Фильтр по указанному приоритету.
- **filter_by_status** — Фильтр по указанному статусу.
- **filter_by_date** — Фильтр по крайней указанной дате. Принимает на вход объект *datetime*.
- **filter_by_days** — Фильтр по крайнему указанному целому количеству дней. Принимает на вход лишь число.

# Обработка задач
`src/executor.py` Объект исполнения задач с workers. В `src/handlers.py` находится контракт и хендлеры для каждого из источника задач.

Для демонстрации разных исходов исполнения при обработке задачи из генератора с описанием "must fail" задача получит статус failed

# Функция main
Доступны 10 команд: `add-task`, `add-many-tasks`, `show-tasks`, `change-task-status`, `executor`, `available-statuses`, `find-task,` `filter-by-priority`, `filter-by-status`, `exit`

Работает с очередью задач. создаются через генератор, ошибки ввода попадаются валидаторами.

# Логгирование
